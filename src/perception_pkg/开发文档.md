# 模块B：感知节点开发文档

**开发环境**：Ubuntu 24.04 + ROS2 Jazzy
**依赖模块**：模块A（mickrobot_description）提供仿真环境与传感器话题

---

## 一、模块功能概述

本模块负责机器人的 **建图（SLAM）** 与 **定位（Localization）**，对外提供统一的感知输出接口，供模块C（导航）和模块E（航点检测）使用。

整体工作流分两阶段：

```
阶段1：SLAM 建图  →  保存地图文件
阶段2：加载地图 + AMCL 定位  →  持续输出实时位姿
```

---

## 二、输入话题（来自模块A）

| 话题 | 消息类型 | 用途 |
|------|----------|------|
| `/scan` | `sensor_msgs/LaserScan` | SLAM 建图 + AMCL 定位的激光数据 |
| `/odom` | `nav_msgs/Odometry` | RTAB-Map 里程计输入（仿真精度等同真值） |
| `/imu/data` | `sensor_msgs/Imu` | 真实模式下 RTAB-Map 使用，仿真模式关闭 |
| `/camera/left/image_raw` | `sensor_msgs/Image` | 真实模式双目 SLAM（仿真模式不使用） |
| `/camera/right/image_raw` | `sensor_msgs/Image` | 真实模式双目 SLAM（仿真模式不使用） |

顶层参数：`use_sim:=true/false`（与模块A保持一致）

---

## 三、输出话题（供模块C / E使用）

| 话题 | 消息类型 | 说明 |
|------|----------|------|
| `/perception/map` | `nav_msgs/OccupancyGrid` | 栅格地图（建图阶段实时更新，定位阶段为静态地图） |
| `/perception/current_pose` | `geometry_msgs/PoseWithCovarianceStamped` | 机器人在 map 帧下的实时位姿，含协方差 |
| `/perception/waypoints` | `geometry_msgs/PoseArray` | 透传模块E发布的图钉航点 |

---

## 四、航点接口（供模块E写入）

模块E将检测到的图钉坐标发布到：

```
话题：/detection/waypoints
类型：geometry_msgs/PoseArray
frame：map
```

本模块自动将其透传至 `/perception/waypoints`，模块C订阅 `/perception/waypoints` 即可获取航点。模块E无需修改本模块代码，只需发布到上述话题。

---

## 五、完整使用流程

### 阶段1：SLAM 建图

```bash
# 终端1：启动仿真（模块A）
ros2 launch mickrobot_description bringup.launch.py use_sim:=true

# 终端2：启动 SLAM 建图
ros2 launch perception_pkg perception.launch.py use_sim:=true mode:=slam

# 终端3：自动建图（方形螺旋路径）
ros2 run perception_pkg patrol_mapper.py
# 或自主探索（激光避障随机游走）
ros2 run perception_pkg auto_explore.py
# 或手动遥控
ros2 run teleop_twist_keyboard teleop_twist_keyboard

# 终端4：地图满意后保存（SLAM 保持运行，不要 Ctrl-C）
ros2 launch perception_pkg save_map.launch.py use_sim:=true
```

保存输出：
- `maps/slam_map.yaml`：地图元数据
- `maps/slam_map.pgm`：占用栅格图像

### 阶段2：定位模式

```bash
# Ctrl-C 终止终端2、3，然后：
ros2 launch perception_pkg perception.launch.py use_sim:=true mode:=localize
```

AMCL 自动从 `(0, 0, 0°)` 初始化（与 Gazebo 出生点一致）。移动机器人几步后定位收敛，`/perception/current_pose` 开始输出有效位姿。

### 定位验证

```bash
# 查看实时位姿（协方差非零说明 AMCL 正常更新）
ros2 topic echo /perception/current_pose --once

# 查看 TF 修正量
ros2 run tf2_ros tf2_echo map odom
```

### RViz 可视化推荐配置

| Display | Topic | 说明 |
|---------|-------|------|
| Map | `/perception/map` | 栅格地图 |
| PoseWithCovariance | `/perception/current_pose` | 机器人位姿椭圆 |
| ParticleCloud（nav2_rviz_plugins）| `/particle_cloud` | AMCL 粒子云，Reliability 设为 Best Effort |
| LaserScan | `/scan` | 实时激光点 |

## 六、关键设计决策与调优说明

### 为何选择 RTAB-Map 而非 Cartographer

仿真环境下（WSL2）`/scan` 实际频率约 1.5Hz（标称 10Hz），Cartographer 对帧率有较高要求，容易因帧率不足导致子图构建失败。RTAB-Map 通过 ICP 逐帧匹配，单帧处理更鲁棒，适合低频激光环境。

### 仿真模式关闭里程计订阅

```yaml
# rtabmap_slam_sim.yaml
use_odometry: false  # 已注释说明：WSL2 里程计时间戳不准
```

Gazebo 差速里程计精度高（无滑移），但 WSL2 下 `/clock` 与系统时钟偶有抖动，导致 RTAB-Map 时间同步失败。改为只用激光 ICP 匹配，规避时间同步问题。

### ICP 参数适配低频雷达

```yaml
Icp/MaxTranslation: "0.3"   # 1.5Hz × 0.2m/s = 每帧最多移动 0.13m，给 2× 余量
Icp/MaxRotation:    "0.7854" # ≈45°，快速转弯时不丢失配准
Icp/CorrespondenceRatio: "0.15"  # 降低要求，长直墙特征少时仍可匹配
```

### 栅格动态清除参数

```yaml
Grid/OccupancyThr: "0.55"  # 低于默认 0.65，旧墙格更易被新视角清除
Grid/ProbHit:      "0.65"  # 低于默认 0.70，避免旧障碍格被过度强化
Grid/ProbMiss:     "0.45"  # 高于默认 0.40，穿越后自由区域更新更快
```

### Gazebo 渲染降负（small_house.world）

已关闭阴影渲染（`cast_shadows=0`、`shadows=0`）并将物理频率从 1000Hz 降至 500Hz，目的是释放 WSL2 CPU，使 `/scan` 频率从 ~1.5Hz 提升，改善建图连续性。

## 七、文件结构

```
B/perception_pkg/
├── launch/
│   ├── perception.launch.py      # 主入口，mode:=slam / localize
│   ├── save_map.launch.py        # 保存 /map 到文件
│   ├── record_bag.launch.py      # 录制 rosbag（用于离线调试）

├── config/
│   ├── rtabmap_slam_sim.yaml     # RTAB-Map 仿真参数
│   ├── rtabmap_slam_real.yaml    # RTAB-Map 真实机器参数
│   ├── amcl_params.yaml          # AMCL 定位参数
│   ├── cartographer_sim.lua      # Cartographer 备用配置（当前未启用）
│   ├── cartographer_real.lua     # Cartographer 备用配置（当前未启用）
│   └── slam_monitor.rviz         # RViz 预设配置
├── scripts/
│   ├── patrol_mapper.py          # 自动建图：方形螺旋路径
│   ├── auto_explore.py           # 自动建图：激光避障随机探索
│   └── odom_to_tf.py             # 里程计转 TF 工具（调试用）
└── maps/
    ├── slam_map.yaml             # 保存的地图元数据
    └── slam_map.pgm              # 保存的地图图像
```

## 八、与其他模块的对接说明

### 对接模块C（导航）

模块C直接订阅以下话题即可，无需修改本模块：

```python
/perception/map          # OccupancyGrid，用于路径规划
/perception/current_pose # PoseWithCovarianceStamped，用于当前位置
/perception/waypoints    # PoseArray，图钉目标点
```

TF 树由本模块维护：`map → odom → base_footprint`，导航节点可直接使用。

三终端导航启动顺序、lifecycle 就绪标志与故障排查见 [`navigation_pkg/开发文档_C.md`](../navigation_pkg/开发文档_C.md) 与根目录 [`README.md`](../../README.md) 模块 C 章节。

### 对接模块E（航点检测）

模块 E（`detection_pkg`）发布：

```python
/detection/waypoints   # geometry_msgs/PoseArray, frame_id=map
/detection/annotated_image  # sensor_msgs/Image, 左目绿框
/detection/markers     # visualization_msgs/MarkerArray
```

本模块 `perception_humble.launch.py` 通过 relay 透传至 `/perception/waypoints`。

**Humble 建图会话**（`humble_sim_slam.launch.py`）每次创建 `maps/map_<时间戳>/`，检测节点通过 `session_dir` 写入 `waypoints.yaml`（**已去重**：默认 0.35 m 半径 + ORB 词袋重识别，每秒刷盘）；`map_snapshot_saver` 写入 `slam_map.*` 与 `initial_pose.yaml`（建图过程中持续更新）。

定位时传入：

```bash
initial_pose_file:=maps/map_<时间戳>/initial_pose.yaml
```

详见根目录 [README.md](../../README.md) 与 [docs/TOPIC_COMMUNICATION.md](../../docs/TOPIC_COMMUNICATION.md)。
