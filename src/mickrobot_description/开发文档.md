# 模块A（v2）：UGV 六轮底盘 URDF 仿真搭建与验证

## 1. 模块总览

本模块基于实体 UGV 机器人的 STL 三维扫描模型，通过 Blender MCP 完成了模型尺寸测量与参数提取，在 **ROS 2 Jazzy** 环境基于 **Gazebo Harmonic (gz-sim)** 重新构建了六轮差速底盘的完整 URDF 仿真模型。

相较于 v1 版本（四轮 mickx4 底盘），本版本的主要升级：

| 升级项 | v1 | v2 |
|--------|----|----|
| 底盘视觉 | SolidWorks 导出的分体 STL | **UGV 实体扫描整体 STL** |
| 驱动布局 | 4 轮差速 | **6 轮差速（4 驱动 + 2 被动中轮）** |
| 摄像头 | 单目 | **双目立体视觉**（基线 120mm） |
| 激光雷达 | 已有 | **2D LiDAR**（360°, 12m 量程） |
| 尺寸精度 | 基于 CAD 图纸 | **基于 Blender 实测（误差 ±2mm）** |

---

## 2. 功能包内容说明 (`mickrobot_description`)

- `urdf/mickrobot_ugv.urdf`：**主 URDF 配置**（六轮版），包含惯性张量、摩擦系数、碰撞几何体、Gazebo 插件。
- `urdf/mickrobot_stl.urdf`：旧版四轮 URDF（保留供参考）。
- `urdf/mesh/ugv01_body.STL`：UGV 实体扫描模型（用于 base_link 视觉渲染）。
- `urdf/mesh/mickx4_*.STL`：旧版分体网格文件（保留）。
- `launch/bringup.launch.py`：**顶层启动文件**，已切换到 `mickrobot_ugv.urdf`。
- `worlds/nav.world`：Gazebo 仿真地图。
- `rviz/urdf_gazebo.rviz`：RViz 预配置界面。

| 参数 | 实测值 | URDF 使用值 |
|------|--------|-------------|

| 质量（底盘） | ~2 kg（估算） | 10 kg（含电池等） |

## 4. URDF 机器人结构

```
base_footprint
  └── base_link (UGV 整体 STL 视觉模型)
        ├── left_front_wheel_joint  → left_front_wheel  (驱动)
        ├── right_front_wheel_joint → right_front_wheel (驱动)
        ├── left_mid_wheel_joint    → left_mid_wheel    (被动)
        ├── right_mid_wheel_joint   → right_mid_wheel   (被动)
        ├── left_back_wheel_joint   → left_back_wheel   (驱动)
        ├── right_back_wheel_joint  → right_back_wheel  (驱动)
        ├── imu_joint      → imu_link
        ├── gnss_joint     → gnss_link
        ├── laser_joint    → laser_link (2D LiDAR)
        ├── camera_left_joint  → camera_left_link  (左目)
        └── camera_right_joint → camera_right_link (右目)
```

### 传感器话题

| 传感器 | ROS 2 话题 | 类型 |
|--------|-----------|------|
| 2D LiDAR | `/scan` | `sensor_msgs/LaserScan` |
| 左目摄像头 | `/camera/left/image_raw` | `sensor_msgs/Image` |
| 右目摄像头 | `/camera/right/image_raw` | `sensor_msgs/Image` |
| IMU | `/imu/data` | `sensor_msgs/Imu` |
| GNSS | `/gps/fix` | `sensor_msgs/NavSatFix` |
| 里程计 | `/odom` | `nav_msgs/Odometry` |
| 控制输入 | `/cmd_vel` | `geometry_msgs/Twist` |

## 5. 与初始框架设计的关键差异说明 (任务 B/C 必读)

为了适配组员实际使用的开发环境并解决一些底层兼容性问题，实际交付的 A 模块对原定《框架.md》进行了几项重大调整，这将**直接影响后续感知 (B) 和导航 (C) 节点的开发**：

### ① 操作系统与 ROS 版本变更
- **原定**：Ubuntu 22.04 + ROS 2 Humble
- **实际采用**：**Ubuntu 24.04 + ROS 2 Jazzy**
- **影响**：请所有后续开发同学确保使用 Ubuntu 24.04 和 ROS 2 Jazzy。在安装后续的 Cartographer 或 Nav2 时，不能照抄网上的 Humble 教程，所有的 `apt install` 必须替换为 `ros-jazzy-cartographer-ros` 或 `ros-jazzy-navigation2`。

### ② 仿真器引擎更替
- **原定**：传统 Gazebo Classic (搭配 `gazebo_ros_pkgs`)
- **实际采用**：**新一代 Gazebo Sim (版本代号 Harmonic)**
- **影响**：底层物理引擎和插件架构已经完全改变。URDF 文件中我们使用的是 `<plugin filename="gz-sim-diff-drive-system">`。传感器数据不再通过传统的 `libgazebo_ros_camera.so` 发布，而是统一通过 `ros_gz_bridge` 在 Launch 文件中桥接到 ROS 2 网络。

### ③ 传感器与底盘参数校准
- **双目摄像头**：基准线根据实际 OAK 相机标定文件配置为 **75.55 mm**。
- **底盘驱动**：为了解决原地打滑问题，摒弃了 4 轮驱动方案，改为 **6 轮全驱 + 各向同性摩擦 (mu1=mu2=0.5)**。
- **IMU 与 GNSS**：IMU 被成功挂载，但由于实际小车没有 GNSS (GPS) 模块，仿真中已将 GNSS 移除。建图和定位模块请**只依赖 LiDAR 和 Odom/IMU 数据**。

## 6. 防坑指南与关键架构设计

### a. 六轮差速驱动策略

采用 **4 轮驱动 + 2 被动中轮** 方案：
- 前后 4 轮通过 `gz::sim::systems::DiffDrive` 插件绑定驱动。
- 中间 2 轮设为 `continuous` 关节（自由旋转），低摩擦系数（μ1=0.8）以减少转向阻力。
- `wheel_separation = 0.18m`，`wheel_radius = 0.04m`。

### b. STL 视觉模型注意事项

- UGV 整体 STL 作为 `base_link` 的 visual mesh，轮子部分是固定在模型中的（不会跟随关节旋转）。
- URDF 中的 6 个独立轮子 link 使用简单圆柱体，可能与 STL 中嵌入的轮子视觉重叠。这是正常现象，不影响仿真物理行为。
- 碰撞几何体使用简化的 box/cylinder，保证仿真性能。

### c. `sim / real` 软硬件一键切换接口

系统保留了完整的仿真与真实本体无缝切换口！
在顶层 Launch 文件中提供了标准指令参数：`use_sim`。

- 如果执行 `use_sim:=true`（默认），则会加载 Gazebo 和全套桥接网络。
- 如果执行 `use_sim:=false`，所有仿真节点都会被屏蔽禁用。

## 6. 运行验证指南

```bash
# 1. 直接编译
cd <你的工作空间>
colcon build --packages-select mickrobot_description
source install/setup.bash

# 2. 启动仿真模式（自动同时弹起 Gazebo 和 RViz2）
ros2 launch mickrobot_description bringup.launch.py use_sim:=true

# 3. 开启另外一个终端用于控制测试
ros2 run teleop_twist_keyboard teleop_twist_keyboard
```

当能用键盘在 Gazebo 里面控制六轮小车，且 RViz 里能看见：
- 双目摄像头画面（左/右）
- 激光雷达 360° 点云
- TF 坐标系正确显示

代表本层的封装交付 100% 成功。
